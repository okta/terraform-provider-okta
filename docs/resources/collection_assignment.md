# Resource: okta_collection_assignment

Manages an assignment of an Okta Governance Collection to a principal (user or group). When a collection is assigned to a principal, POLICY grants are automatically created in the background, providing access to all resources and entitlements in the collection.

~> **Note:** This resource requires Okta Governance (Okta Identity Governance - OIG) to be enabled in your Okta organization.

~> **Important:** POLICY grants created by collection assignments cannot be managed directly. They are automatically created and removed based on collection assignments.

## Example Usage

### Basic Assignment

```terraform
resource "okta_collection_assignment" "engineering_team" {
  collection_id   = okta_collection.slack_permissions.id
  principal_id    = okta_group.engineers.id
  principal_type  = "OKTA_GROUP"
  actor          = "ADMIN"
}
```

### Assignment with Expiration

```terraform
resource "okta_collection_assignment" "temporary_access" {
  collection_id   = okta_collection.project_x.id
  principal_id    = okta_user.contractor.id
  principal_type  = "OKTA_USER"
  actor          = "API"
  expiration_time = "2024-12-31T23:59:59Z"
  time_zone       = "America/Los_Angeles"
}
```

## Schema

### Required

- `collection_id` (String) The ID of the collection to assign.
- `principal_id` (String) The ID of the principal (user ID or group ID).
- `principal_type` (String) The type of principal. Valid values: `OKTA_USER`, `OKTA_GROUP`.

### Optional

- `actor` (String) The actor making the assignment. Valid values: `ACCESS_REQUEST`, `ADMIN`, `API`, `NONE`. Default: `API`.
- `expiration_time` (String) The date/time when the assignment expires (ISO 8601 format).
- `time_zone` (String) The time zone in IANA format for the expiration date.

### Read-Only

- `id` (String) The assignment ID.
- `assignment_type` (String) The type of assignment (e.g., INDIVIDUAL).
